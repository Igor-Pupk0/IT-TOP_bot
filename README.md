### Чо по доп функционалу?
1) Есть админка (команда /skibidi_admin)
2) Есть уведомления о просрочке дз
3) В лидербордах есть фотокарточки студентов
4) В отзывах есть более точное время написания, в активностях так же
5) Автоматическая оценка пар
6) В сдавании дз лимит на текстовый ответ расширен, не 500 а 1000 символов (API может принять и больше) + минимального лимита в 5 символов нет

### Как развернуть?
1) Скопировать файл `.env_example` и переименовать его в `.env`
2) Заполнить `.env` файл
    - BOT_TOKEN - токен от тг бота
    
    - POSTGRES_DB_NAME - название базы данных postgres, в которой будут храниться данные
    - POSTGRES_PASSWORD - пароль от нее
    - DEV_TELEGRAM_ID - ID телеграмм акаунта разработчика (нужен для доступа к админке)
    
    - BOT_ENV - режим запуска бота, всего есть 2 режима - **dev** и **prod**, единственное их отличие в способе получения данных о сообщениях:
        - **dev** использует infinity_pooling, для запуска на пк
        - **prod** использует вебхуки, для сервера

    Параметры необходимые для сервера (или же режима prod):
    - WEBHOOK_DOMAIN - домен для вебхука или же домен сервера
    - WEBHOOK_ENDPOINT - эндпоинт самого вебхука

3) Если используем **prod** - нужен сертификат lets encrypt, его файлы кидаем в папку `certs`
4) Запускаем бота командой:
    
    *Для prod:*
```bash
docker compose --profile prod up -d
```
    *Для dev:*
```bash
docker compose up -d
```

#### Важное примечание: изза внешнего сервиса на который загружаются html файлы с оценками, который не работает без впн, сами оценки могут не загружатся
