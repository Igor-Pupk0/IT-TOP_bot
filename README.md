# IT-TOP Bot

## Описание Проекта

IT-TOP Bot - это Telegram-бот, разработанный для упрощения и улучшения взаимодействия студентов колледжа с официальным сайтом. Он решает ключевые проблемы, такие как нестабильность работы сайта, частые сбои и необходимость постоянной повторной авторизации, предоставляя надежный и удобный интерфейс для доступа к важной информации.

## Основные Возможности

Бот предоставляет широкий спектр функций для студентов:

*   **Журнал и Оценки:** Просмотр текущих оценок и статистики.
*   **Расписание:** Доступ к актуальному расписанию занятий.
*   **Домашние Задания:** Получение, отправка и отслеживание статуса домашних заданий. Расширен лимит на текстовый ответ до 1000 символов, без минимального ограничения в 5 символов.
*   **Уведомления:** Получение дедлайнов по домашним заданиям, включая уведомления о просроченных ДЗ.
*   **Профиль:** Просмотр личной информации и настроек.
*   **Лидерборды:** Просмотр рейтингов студентов с их фотокарточками.
*   **Активность:** Отслеживание активности с более точным временем событий.
*   **Отзывы:** Возможность просмотра отзывов с более точным временем написания.
*   **Автоматическая оценка пар:** Система для автоматической оценки занятий.
*   **Оценки:** Просмотра оценок с темной темой.
*   **Административные функции:** Доступ к панели администратора через команду `/skibidi_admin` для разработчика.
*   **Дополнительные функции:** Экзамены, магазин и меню.
*   **Логирование:** Логирование ошибок и действий пользователей

## Используемые Технологии

Проект построен на современном стеке технологий, обеспечивающем его стабильность, производительность и масштабируемость:

*   **Python:** Основной язык разработки.
*   **Flask:** Для вебхуков.
*   **Docker:** Контейнеризация приложения для легкой изоляции и развертывания.
*   **Nginx:** Высокопроизводительный веб-сервер, используемый для тех же вебхуков.
*   **PostgreSQL:** Надежная реляционная база данных для хранения информации.
*   **Telebot:** Фреймворк для разработки Telegram-ботов на Python.

## Установка и Запуск

Для запуска проекта необходимо иметь установленный Docker и Docker Compose.

1.  **Клонируйте репозиторий:**
    ```bash
    git clone https://github.com/Igor-Pupk0/IT-TOP_bot.git
    cd IT-TOP_bot
    ```

2.  **Настройте переменные окружения:**
    Создайте файл `.env` в корне проекта, скопировав `.env_example` и заполнив его необходимыми данными.

    ```bash
    cp .env_example .env
    # Отредактируйте .env файл, заполнив следующие переменные:
    ```

    *   `BOT_TOKEN`: Токен вашего Telegram бота, полученный от BotFather.
    *   `POSTGRES_DB_NAME`: Имя базы данных PostgreSQL для хранения данных бота.
    *   `POSTGRES_PASSWORD`: Пароль для доступа к базе данных PostgreSQL.
    *   `DEV_TELEGRAM_ID`: Telegram ID разработчика, необходимый для доступа к административным функциям (например, `/skibidi_admin`).
    *   `BOT_ENV`: Режим запуска бота. Возможные значения:
        *   `dev`: Использует `infinity_polling` для запуска на локальном ПК.
        *   `prod`: Использует вебхуки для запуска на сервере.

    **Параметры, необходимые для режима `prod`:**

    *   `WEBHOOK_DOMAIN`: Доменное имя вашего сервера, используемое для вебхуков.
    *   `WEBHOOK_ENDPOINT`: Эндпоинт вебхука на вашем сервере (например, `/webhook`).

    **Важно для `prod` режима:** Если вы используете режим `prod`, необходимо наличие сертификата Let's Encrypt (или аналогичного). Файлы сертификата (например, `fullchain.pem` и `privkey.pem`) должны быть помещены в папку `certs/` в корне проекта.

3.  **Разверните приложение с помощью Docker Compose:**

    *   **Для режима `prod` (на сервере с вебхуками):**
        ```bash
        docker compose --profile prod up -d
        ```
    *   **Для режима `dev` (локально с polling):**
        ```bash
        docker compose up -d
        ```
    Эти команды соберут Docker-образы и запустят все сервисы (бот, API, Nginx, PostgreSQL) в фоновом режиме в соответствии с выбранным профилем.

## Использование

После успешного запуска бота, вы можете взаимодействовать с ним через Telegram. Найдите бота по его имени и используйте команду `/start` для начала работы.

**Важное примечание:** Из-за зависимости от внешнего сервиса на который загружаются html файлы с оценками, который не работает без vpn, сами оценки могут не загружаться
